Dev Bootcamp - Final Project - Feb 2015

What is it?

Hummingbird was created with a purpose of scheduling SMS texts so they could be sent at the right time. Our idea was to have the backend API always up and running so that frontend users could connect to it. The best way to envision this idea was through decoupled architecture. By decoupling the web app, the backend could actively batch up messages to be sent while the frontend allowed users to access the application on their devices to schedule their messages.

Tech Stack
Frontend: Ionic Framework with AngularJS
Backend: Rails with PostgreSQL
Decoupled Archiecture
Twilio API

Dev Bootcamp - Survey Monkey Clone - January 2015

What is it?

Our project, Survey Gorilla Ape Monkey Chimp, is a clone of SurveyMonkey. While capturing the RESTful routes aspect of web development, we also wanted to focus on the UX/UI experience for our application. In order to accomplish this, we learned about AJAX and jQuery libraries in our application. Tranversing the HTML DOM with jQuery saved us a ton of time versus having to do it with vanilla JavaScript. AJAX allowed us to asynchronously retrieve responses from the server without actively reloading the website thus effectively making it a single page application. The important lesson learned from this project was to strive to meet your goals and to build goals on top of the goals that were met.

Tech Stack
Frontend: Embedded Ruby and Bootstrap Grid System
Backend: Sinatra
MVC Archiecture
Includes jQuery and AJAX features

Dev Bootcamp - !Twitter Clone Project - Jan 2015

What is it?

!Twitter was group project that had to be built and presented in two days. Upon reaching our MVP milestone, we would throw on more and more features to make the project more usable and attractive. Utilizing the web development skills we learned earlier in the week, our goal was to create a clone of Twitter using Ruby and Sinatra. Teams were randomized so that we would work with others that we may or may not have paired programmed with before. Our group had different skill levels, different ways of approaching tasks, different comfort zones, and etc. This lead to many struggles and new experiences I had never faced before. The most important takeaway from this project was understanding and filling in the technical gaps in the group as a team. Just ask me about it!

Tech Stack
Frontend: Embedded Ruby
Backend: Sinatra
MVC Archiecture
